# Admin Setup

Admins make the decisions that let Valet work safely for a team: who can join, which tools are connected, where messages can route, and which actions require approval.

## Setup checklist

<Steps>
  <Step title="Configure access">
    Decide who can sign in, which domains or invites are allowed, and who should have admin privileges.
  </Step>
  <Step title="Connect integrations">
    Enable the systems Valet should use, such as GitHub, Slack, Google Workspace, Linear, Notion, Stripe, or custom MCP services.
  </Step>
  <Step title="Set approval defaults">
    Decide which services, actions, or risk levels can run automatically, which require approval, and which are denied.
  </Step>
  <Step title="Choose channels">
    Decide where users can talk to Valet and which surfaces can route into personal or organization context.
  </Step>
  <Step title="Define shared behavior">
    Create shared personas, skills, workflows, and memory guidance for recurring team work.
  </Step>
</Steps>

## Recommended starting point

Start narrow:

- Enable one or two integrations.
- Require approval for write actions.
- Use the web app for early sessions.
- Add Slack or Telegram once routing rules are clear.
- Create one shared workflow only after the team has repeated the work manually.

## What to revisit

After teams start using Valet, review:

- Which approvals are noisy.
- Which actions should be blocked.
- Which workflows are worth standardizing.
- Which memories or personas should be shared.
- Which channels need tighter routing rules.
