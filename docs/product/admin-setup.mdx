Admins prepare Valet so a team can use it safely.

## Setup checklist

<Steps>
  <Step title="Choose who can join">
    Decide which teammates can use Valet and who should be an admin.
  </Step>
  <Step title="Connect the first tools">
    Start with the systems needed for the team's first real use case, such as Slack, GitHub, Linear, Gmail, Google Workspace, or Notion.
  </Step>
  <Step title="Set approval rules">
    Decide which actions Valet can do on its own and which actions should ask first.
  </Step>
  <Step title="Choose channels">
    Decide where people can talk to Valet and where Valet should send updates.
  </Step>
  <Step title="Create shared skills and workflows">
    Capture repeated work as skills and workflows so the team does not reinvent the same process.
  </Step>
</Steps>

## Start narrow

For the first rollout:

- Enable one or two tools.
- Require approval for write actions.
- Start in the web app.
- Add Slack or Telegram once the team knows where updates should go.
- Create one workflow only after the team has done the work manually.

## Review after real use

After a few tasks, review:

- Which approvals were helpful.
- Which approvals were noisy.
- Which tools Valet was missing.
- Which repeated tasks should become workflows.
- Which instructions should become shared skills.
- Which channels need tighter rules.
