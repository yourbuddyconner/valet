# Design Language

Valet should read as trustworthy infrastructure with a little personality. The docs site is calm, capable, and plain-spoken: serious work first, friendly valet second.

The implementation rule is simple: use Mintlify for the docs system, then add custom structure only where it creates real product polish.

## Theme

Use `mint` as the docs theme. Turnkey's docs prove the theme is not the main source of polish; the effect comes from information architecture, a custom homepage, reusable MDX snippets, deliberate assets, and restrained CSS.

Keep the theme choice boring and durable. Use `docs.json` for site-wide configuration, `styles.css` for scoped custom homepage and card styles, and Mintlify built-ins for normal documentation pages.

## Information architecture

The docs should have five top-level tabs:

| Tab | Purpose |
| --- | --- |
| Home | A custom, general-audience landing page. |
| Use cases | What Valet does, organized by job to be done. |
| Product docs | Concepts, quickstart, and guides for all users. |
| Engineering | Sandboxes, sessions, policies, orchestration, security, setup, and deployment. |
| API & MCP | Endpoints, SDKs, and the MCP server. |

Use `docs/index.mdx` with `mode: "custom"` for the homepage. Use `docs/styles.css` for `valet-*` classes only. Use `docs/snippets/` for reusable cards and page building blocks.

The root stays conceptual. Depth lives one level down. An operator should never hit a sandbox policy schema on an overview page.

## Color

| Token | Hex | Use |
| --- | --- | --- |
| Signal Blue | `#1747E6` | Primary links, key actions, selected states, and the uniform. |
| Deep Blue | `#0E2DB3` | Hover and pressed states. |
| Navy | `#0A1340` | Dark code surfaces and deep backgrounds. |
| Star Gold | `#F2C84A` | Accent only: star, tips, highlights. |
| Cream | `#F4EFE2` | Main canvas. Never use pure white as the dominant page background. |
| Paper | `#FAF7EC` | Raised content surfaces. |
| Ink | `#0D1226` | Primary text. |
| Muted | `#6A6E85` | Secondary text and metadata. |
| Line | `#D6CFB9` | Hairline borders and dividers. |

Aim for mostly neutral surfaces, a small amount of blue for direction, and gold only as seasoning.

## Typography

Use Geist for display, body, and interface copy when the final font assets are available. Use Geist Mono for labels, metadata, and code. Use Instrument Serif Italic as a rare accent phrase in headlines.

Use short mono-style labels for eyebrows and metadata:

```text
SESSIONS
WORKSTATIONS
POLICY
MCP
```

Use one italic serif flourish per headline at most. Set it in Signal Blue.

## Motifs

Use these motifs consistently:

- **Gold star**: the lookup mark and accent. Do not scatter it through every card.
- **Blueprint grid**: only on blue fields or technical diagrams, and only when it helps suggest "under the hood".
- **Mono eyebrows**: short, uppercase labels for sections and metadata.
- **Hairline cards**: paper-fill cards with quiet borders and compact content.
- **Pill chips**: tags, channels, capabilities, and selected state.
- **Dark code surface**: one navy code surface with clear syntax contrast.

## Mascot

The Valet mascot is a friendly uniformed guide. He should feel competent and eager, never goofy. Use him as a small contextual cue, not a decorative pattern.

Recommended mapping:

| Asset | Intent |
| --- | --- |
| `wave.png` | Welcome or first-run orientation. |
| `pointing-right.png` | "This way" navigation or next step. |
| `thinking.png` | Notes, tips, or questions. |
| `writing.png` | Documentation, memory, or workflow authoring. |
| `thumbs-up.png` | Success or completion. |
| `stop.png` / `no.png` | Warnings, denials, or do-not guidance. |
| `workspace.png` | Hosted workspaces. |
| `child-sessions.png` | Delegation and orchestrator child sessions. |
| `personas.png` | Persona and skill configuration. |

Use one mascot image per page or callout. Do not recolor, stretch, rotate, or place the mascot on busy imagery.

## Voice

Valet copy should be clear, concise, helpful, and unhurried.

Do:

- Lead with the answer, then give the detail.
- Use short sentences and active voice.
- Keep technical pages precise.
- Let the mascot carry personality in visual moments.

Do not:

- Over-explain simple workflows.
- Turn every note into a joke.
- Hide important technical constraints behind marketing copy.
- Make destructive or approval-sensitive actions sound casual.

## Homepage

The homepage is a custom product surface, not a stock themed index.

Required sections:

1. Hero: headline, CTAs, search, waving mascot.
2. Use-case grid: three or four jobs to be done.
3. How Valet works: request, context, workspace, tools, approval, result.
4. Integrations: connected tools.
5. Start building: CTA into the quickstart.

Hero rules:

- Headline: "Your AI coworker, documented." with one blue serif accent.
- Primary CTA: product quickstart.
- Secondary CTA: concepts.
- Search: prominent "search or ask" affordance.
- Mascot: waving, right of hero, on a blue disc.
- Asset direction: grid field plus mascot cut-out. No photos.

## Reusable snippets

The snippet kit should cover:

| Snippet | Use |
| --- | --- |
| Feature card | One capability in a line. |
| Use-case card | A job to be done. |
| Persona card | Behavior and tools preset. |
| Integration card | A connected service. |
| Reference card | Jump to an endpoint or deep technical page. |
| Spotlight card | Featured guide or accent variant. |

Keep snippets small and consistent. Do not build one-off cards on individual pages.

## Page templates

Every page should map to one of these templates:

| Template | Use |
| --- | --- |
| Overview | Concept plus links out. Buyer's entry. |
| Concept | Prose-first explanation. |
| Quickstart | Numbered steps to first value. |
| Integration guide | Setup plus behavior and safety notes. |
| API / MCP reference | Params, examples, responses, connection details. |
| Troubleshooting | Symptom, cause, fix. |
| Use case | Story-led, outcome-first, cards. |

## Mintlify usage

Prefer Mintlify built-ins:

- Use `Note`, `Tip`, `Check`, `Warning`, and `Danger` for semantic callouts.
- Use `CardGroup` and `Card` for wayfinding.
- Use `Steps` for setup flows.
- Use `CodeGroup` for language examples.
- Use `ParamField` and `ResponseField` for API reference pages.

Use custom CSS for:

- The homepage shell in `docs/index.mdx`.
- Shared card snippets in `docs/snippets/`.
- Light/dark image switching when the final brand kit includes paired assets.

Do not use custom CSS to restyle every docs page. If a built-in Mintlify component can express the page, use it.

## Asset folders

Use these locations for the final brand kit:

| Folder | Use |
| --- | --- |
| `docs/images/brand/` | Logo alternates, social image, palette references, and final brand-kit notes. |
| `docs/images/home/` | Hero art, grid artwork, and homepage-only illustrations. |
| `docs/images/icons/` | Tab, feature, and use-case icons. |
| `docs/images/characters/` | Mascot poses and character cut-outs. |
