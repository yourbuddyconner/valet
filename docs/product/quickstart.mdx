# Quickstart

Start with one real piece of work. Valet is easiest to understand when it has a task, the context it needs, and clear rules for what it can do.

## Before you start

You need:

- A Valet account.
- At least one connected channel or the web app.
- The integrations Valet should use for the task.
- A clear sense of which actions should require approval.

## Start your first session

<Steps>
  <Step title="Open Valet">
    Sign in and open the web app. The web app is the safest place to start because you can see the conversation, work status, approvals, and connected tools in one place.
  </Step>
  <Step title="Describe the task">
    Ask for a specific outcome. For example: "Summarize the open Linear issues assigned to me and draft a plan for the three that look most urgent."
  </Step>
  <Step title="Connect context">
    If Valet needs a tool it cannot reach yet, connect the relevant integration or ask an admin to enable it.
  </Step>
  <Step title="Review approvals">
    When Valet wants to take a sensitive action, review the prompt and choose whether to allow it once, allow it for the session, always allow that exact action, or cancel.
  </Step>
  <Step title="Use the result">
    Read the final summary, inspect what changed, and continue the session if follow-up work is needed.
  </Step>
</Steps>

## Good first tasks

Good first tasks are clear, bounded, and easy to inspect:

- Summarize a set of issues, pull requests, messages, or documents.
- Draft a follow-up plan from information in connected tools.
- Prepare a checklist for a recurring workflow.
- Research account context before a meeting.
- Triage a small queue and ask for approval before writing updates.

## Avoid as first tasks

Avoid tasks that are too broad or hard to verify:

- "Fix everything."
- "Automate our whole process."
- "Delete or modify production data without asking."
- "Make decisions that should belong to a human owner."
