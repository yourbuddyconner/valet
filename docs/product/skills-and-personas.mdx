# Personas and Skills

Personas and skills shape how Valet behaves for a task.

A **persona** is a behavior preset. A **skill** is a specialized way of working. Together, they help Valet act consistently across sessions and workflows.

## Personas

Use personas when different work needs different behavior:

- A PR reviewer should be precise, skeptical, and code-focused.
- A customer workflow assistant should be careful, concise, and approval-aware.
- An operations assistant should summarize clearly and surface risks early.
- An incident scribe should preserve timelines and decisions.

Personas can include instructions, tone, expected outputs, preferred tools, and attached skills.

## Skills

Skills describe how Valet should do a kind of work. A skill might cover:

- How to use a browser.
- How to work with GitHub.
- How to prepare a workflow.
- How to search memory.
- How to draft customer follow-up.
- How to use a specific connected system.

Skills should be concrete. They are most useful when they describe steps, constraints, and quality bars.

## Choosing the right behavior

Start with the default persona for general work. Add or create a persona when:

- The task has a repeatable role.
- The output format matters.
- The agent needs a consistent tone.
- The task needs a specialized tool sequence.
- A team wants shared behavior across many users.

## Admin guidance

Admins should keep shared personas and skills small, named clearly, and tied to real work. A long list of overlapping presets makes Valet harder to operate.
