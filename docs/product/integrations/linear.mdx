## Connection summary

| Field | Value |
| --- | --- |
| Service slug | `linear` |
| Display name | Linear |
| Auth type | `oauth2` |
| Supported entities | `issues`, `projects`, `teams`, `comments` |
| OAuth scopes | `write` |
| MCP server URL | `https://mcp.linear.app/mcp` |
| Default action risk | `medium` |

## Setup

OAuth 2.0 — the user connects this integration through Valet's connection screen, which redirects to the provider's OAuth consent flow and stores the resulting access token in the per-user credential store.

See [OAuth setup](/oauth-setup) for environment variables and connection flow.

## Upstream documentation

This integration proxies the Linear MCP server. For the full, current list of available actions, see the [Linear MCP documentation](https://linear.app/docs/mcp).

## Actions

Actions for Linear are proxied through a remote [Model Context Protocol](https://modelcontextprotocol.io) server at `https://mcp.linear.app/mcp`. The exact list of actions, their parameters, and behavior are owned by the upstream MCP server — Valet enumerates them at runtime when the integration is connected.

Default risk level applied to Linear actions: `medium`. Individual actions may carry their own risk level from the upstream server.

Use `list_tools` with `service: linear` from an agent session to see the live action list, parameter schemas, and per-action risk levels.
