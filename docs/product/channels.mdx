# Channels

Channels are the places where people talk to Valet. A task can start in the web app, Slack, Telegram, a workflow, or another connected surface, then return updates in the right place.

## Web app

The web app is the primary place to inspect work. It shows sessions, progress, approvals, files, workflows, integrations, and settings.

Start in the web app when a task is new, sensitive, or needs close review.

## Slack

Slack is useful for lightweight delegation and updates. A user can ask Valet to help from a direct message or another supported Slack surface, depending on how the organization configures routing.

Use Slack for:

- Quick requests.
- Status checks.
- Approval prompts.
- Follow-up from workflows.

## Telegram

Telegram can be useful for personal or mobile workflows. It can send text and supported media into Valet and receive replies or approval prompts.

## Channel routing

Valet should route a channel message to the right place:

- A bound session if the conversation already belongs to a task.
- The user's orchestrator if it is a new personal request.
- An organization orchestrator or workflow when configured.
- No action if the surface is not allowed.

## Channel safety

Channels can carry sensitive context. Admins should decide:

- Which channels are enabled.
- Which surfaces can reach personal memory.
- Which messages can start sessions.
- Which approval prompts can be resolved outside the web app.
