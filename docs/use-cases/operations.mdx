title: Operations
description: Use Valet for scheduled checks, channel-driven requests, internal workflows, and operational handoffs.
---

Valet can handle recurring operational work when it has the right connected tools, instructions, and approval rules. It can receive requests, gather information, run checklists, and send the result back to the right channel.

Typical work includes:

- Running scheduled checks and posting summaries.
- Responding to Slack or Telegram requests.
- Gathering information from connected tools before taking action.
- Preparing drafts, reports, or follow-up tasks.
- Waiting for approval before actions that change external systems.
- Preserving memory about preferences and recurring procedures.

## Product surfaces involved

- [Workflows](/product/workflows) define recurring or event-driven work.
- [Channels](/product/channels) decide where requests and replies happen.
- [Memory](/product/memory) keeps useful team instructions available over time.
- [Approvals](/product/approvals) keep control explicit.
- [Connect tools](/product/tools) defines what Valet can reach.

<Note>
  Operations use cases are usually less about code and more about remembered procedures, clear channels, and safe tool use.
</Note>
