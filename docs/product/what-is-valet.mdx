# What is Valet?

Valet is an AI coworker with its own workspace. You can ask it to take on work, connect it to the tools your team already uses, and let it continue in the background while you stay in control.

Valet is not just a chatbot. It has a place to work, context it can remember, tools it can use, and rules that decide when it must ask before acting.

## What Valet can do

Valet can help with work that crosses tools, messages, and follow-up steps:

- Answer questions with durable context from earlier work.
- Start and manage tasks from the web, Slack, Telegram, schedules, or webhooks.
- Use connected tools such as GitHub, Linear, Gmail, Google Workspace, Notion, Slack, Stripe, Sentry, and other integrations.
- Run repeatable workflows with human approval points.
- Remember preferences, procedures, and recurring context.
- Report progress back in the channel where the work started.

For engineering teams, Valet can also work inside a development workspace. Those details live in the [Engineering reference](/architecture).

## The product model

Valet has a few concepts that matter to every reader:

| Concept | What it means |
| --- | --- |
| Workspace | The hosted place where Valet does the work for a task. |
| Task session | A focused unit of work with its own conversation, context, participants, files, approvals, and activity. |
| Orchestrator | A persistent coordinator that remembers context and routes work over time. |
| Workflow | A repeatable process that can run manually, on a schedule, or from an external event. |
| Integration | A connected tool or system Valet can use when policy allows it. |
| Approval | A human decision point before Valet performs sensitive actions. |

## Who uses Valet

Different teams can use the same product model in different ways:

- **Operators** can ask Valet to run checks, triage messages, prepare summaries, and handle internal handoffs.
- **Engineering teams** can delegate repository work, PR review, issue triage, and QA loops.
- **Customer-facing teams** can gather account context, draft follow-ups, and create tasks across connected systems.
- **Admins** can configure access, integrations, policies, channels, and approval defaults.
