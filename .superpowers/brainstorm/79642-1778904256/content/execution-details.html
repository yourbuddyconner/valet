Tradeoffs to call out
  • Live update mechanism: the spec lists "real-time step progress" as NOT IMPLEMENTED — completion reports all steps at once. MVP would poll /api/executions/:id/steps every 2s. Real-time via EventBus is a follow-up.
  • Approval gate state: when a step is waiting_approval, the node turns orange and a sticky "Approve / Deny" action surfaces in the header. (Not shown — easy to add.)
  • Failed run: failed step turns red, error message inline, "Retry from here" action (if engine supports it). Currently engine only does full replay; per-step retry would be new work.
  • Output preview: the right-side panel shows a small slice of variables; a "View full output" link expands the step output JSON (especially useful for bash stdout).
