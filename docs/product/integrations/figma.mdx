Pull Figma files, projects, and component metadata for design context.

## Connection summary

| Field | Value |
| --- | --- |
| Service slug | `figma` |
| Display name | Figma |
| Auth type | `oauth2` |
| Supported entities | `files`, `projects`, `components` |
| MCP server URL | `https://mcp.figma.com/mcp` |
| Default action risk | `medium` |

## Setup

OAuth 2.0 — the user connects this integration through Valet's connection screen, which redirects to the provider's OAuth consent flow and stores the resulting access token in the per-user credential store.

Connect from the **Integrations** page in the sidebar — click **Connect Integration**, pick the service, and approve the OAuth consent screen. (Self-hosting Valet? [OAuth setup](/oauth-setup) covers provider credential configuration.)

## Upstream documentation

This integration proxies the Figma MCP server. For the full, current list of available actions, see the [Figma MCP server documentation](https://help.figma.com/hc/en-us/articles/32132100833559-Guide-to-the-Figma-MCP-server).

## Actions

Actions for Figma are proxied through a remote [Model Context Protocol](https://modelcontextprotocol.io) server at `https://mcp.figma.com/mcp`. The exact list of actions, their parameters, and behavior are owned by the upstream MCP server — Valet enumerates them at runtime when the integration is connected.

Default risk level applied to Figma actions: `medium`. Individual actions may carry their own risk level from the upstream server.

Use `list_tools` with `service: figma` from an agent session to see the live action list, parameter schemas, and per-action risk levels.
