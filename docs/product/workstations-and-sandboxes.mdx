# Workspaces

Valet has a workspace for the work it does. That workspace gives the agent a durable place to gather context, use tools, keep files, run workflow steps, and show progress.

For engineering tasks, the workspace can include development tools. For operations and customer workflows, it is still useful as a focused place where Valet can keep task context and interact with connected systems.

## Why workspaces matter

Workspaces make Valet more useful than a stateless assistant:

- The work has a place to continue from.
- Context can survive between messages.
- Actions can be reviewed in one session.
- Files, drafts, logs, and tool output can stay attached to the task.
- Long-running work can pause and resume.

## What users see

Depending on the task and permissions, users may see:

- The session conversation.
- Status and progress updates.
- Approval prompts.
- Files or drafts produced by the task.
- Development surfaces for engineering work.
- Tool output and summaries.

## What admins decide

Admins and team owners decide what workspaces can access:

- Which integrations are available.
- Which repositories or external systems are in scope.
- Which actions require approval.
- Which users can view or collaborate on sessions.
- Whether a workflow can run automatically or must wait for review.

## Good workspace habits

- Give Valet enough context to know what success looks like.
- Connect the tools it needs before asking for a multi-system task.
- Keep sensitive work in private sessions unless collaboration is required.
- Use approvals for actions that write to external systems.
