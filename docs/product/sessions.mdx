# Tasks and Sessions

A session is the place where Valet does a task. It keeps the conversation, status, participants, approvals, connected context, files, and final result together.

For users, the important idea is simple: a session is a work thread with a workspace behind it.

## When sessions are created

Sessions can start in several ways:

- A user starts a task from the web app.
- A message arrives from a connected channel.
- A workflow runs manually.
- A schedule starts recurring work.
- A webhook sends Valet an event.
- An orchestrator delegates part of a larger task to a child session.

## What a session shows

A good session should make the work inspectable:

- What Valet was asked to do.
- Current status and recent progress.
- The messages exchanged with the agent.
- Any approvals waiting on a human.
- Tools or integrations used.
- Files, drafts, or external records changed.
- The final summary and suggested next steps.

## Session types

| Type | Product meaning |
| --- | --- |
| Manual | A user explicitly asked Valet to do something. |
| Scheduled | A recurring check or workflow started at a configured time. |
| Triggered | An external event, such as a webhook, started the work. |
| Workflow | A repeatable process ran with defined steps. |
| Child | A larger task delegated a focused subtask to a separate session. |
| Orchestrator | A persistent coordinator session for a user or organization. |

## Collaboration

Sessions can be private or shared depending on organization settings. A participant may be able to view, collaborate, or own the session.

Use shared sessions when work needs review, handoff, or multiplayer context. Keep sensitive sessions private when they involve personal memory, credentials, customer data, or approval-sensitive actions.

## Sleep and resume

Some sessions can pause when inactive and resume when needed. From a product perspective, this means Valet can keep the history and context of the work without keeping every workspace active forever.

If a session is asleep and a new message arrives, Valet can wake it and continue from the saved context.
