Skills and personas help Valet behave consistently.

- A **skill** teaches Valet how to do a specific kind of work.
- A **persona** shapes how Valet should behave for a role.

<MascotNote
  title="Note / tip"
  heading="Start with the work"
  image="/images/characters/thinking.png"
  alt="Valet thinking through the best setup"
>
  Start with the work, not the label. If a job has a repeatable process, write a skill. If work needs a consistent role or tone, create a persona.
</MascotNote>

## Create a skill

Use a skill when you want Valet to follow the same approach every time.

<Steps>
  <Step title="Name the work">
    Choose a specific name, such as "Review a pull request", "Prepare customer follow-up", or "Summarize a weekly queue."
  </Step>
  <Step title="Write the steps">
    Describe what Valet should check first, what tools it should use, what output it should produce, and what it should avoid.
  </Step>
  <Step title="Add quality rules">
    Include the standards that make the work good: tone, evidence, formatting, approval points, or review criteria.
  </Step>
  <Step title="Attach it where it belongs">
    Attach the skill to a persona, or set it as an organization default skill so it loads for sessions that have no persona.
  </Step>
  <Step title="Test it on real work">
    Run a small session and revise the skill when Valet misses a step or asks for unclear details.
  </Step>
</Steps>

## What a good skill includes

```text
Skill: Review a pull request

Use when:
- A user asks Valet to review a PR before merge.

Steps:
- Read the PR summary and changed files.
- Check for correctness, missing tests, risky behavior, and unclear code.
- Prioritize bugs and regressions over style comments.
- Write concise findings with file references.

Approval:
- Ask before posting comments publicly.
```

Good skills are short, concrete, and tied to real work. If a skill becomes long or vague, split it.

Use a persona when Valet needs to act in a recognizable role.

## What a persona should define

Include:

- The role Valet is playing.
- The tone it should use.
- The kinds of work it should accept.
- The skills it should use.
- What it should ask before doing.
- What a good final answer looks like.

Avoid creating many overlapping personas. A small set of clear roles is easier for users to choose and easier for admins to maintain.
