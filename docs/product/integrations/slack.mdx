Read channels, post messages, look up users, and route Slack-channel conversations.

## Connection summary

| Field | Value |
| --- | --- |
| Service slug | `slack` |
| Display name | Slack |

## Setup

Bot token — an admin pastes a bot token issued by the provider during connection setup. The token is stored in the per-org credential store and reused across users.

| `text` | string | Yes | Message text |

### `slack.dm_user`

| Param | Type | Required | Description |
| --- | --- | --- | --- |
| `user` | string | Yes | User ID (U...) |

### `slack.send_message`

Post a message to a Slack channel or thread. Returns `ts` (message timestamp) and `channel` — save `ts` to thread follow-up messages under this one via `thread_ts`.

**When to use `slack.send_message` vs `channel_reply`:**

- Use `slack.send_message` to post to any channel the bot has joined, or to start/continue a thread on any message. This is the right tool for proactive posts, scheduled reports, and multi-message threads.
- Use `channel_reply` when you want to reply on the exact channel a user wrote *from* (indicated by the `[via slack | chatId: ...]` prefix in the incoming message). `channel_reply` is route-bound to the originating conversation; `slack.send_message` works for any target.

**Worked example — post a header, then thread bullets under it:**

```
# Step 1: post the header
call_tool("slack.send_message", { channel: "#proj-valet", text: "*Weekly status*" })
→ { ok: true, ts: "1780887543.189519", channel: "C0..." }

# Step 2: thread bullets under the header using the returned ts
call_tool("slack.send_message", { channel: "C0...", text: "• Item 1\n• Item 2\n• Item 3", thread_ts: "1780887543.189519" })
```

| Param | Type | Required | Description |
| --- | --- | --- | --- |
| `channel` | string | Yes | Channel ID (`C...`) or channel name with `#` prefix (e.g. `#proj-valet`). Use `list_channels` to find IDs. |
| `text` | string | Yes | Message body. Supports Slack mrkdwn: bold `*text*`, italic `_text_`, code `` `code` ``, links `<url\|label>`. |
| `thread_ts` | string | No | Post as a threaded reply. Use the `ts` from a previous `send_message` response (e.g. `"1780887543.189519"`). |
| `blocks` | string | No | Block Kit JSON array as a string for rich formatting. When provided, `text` is used as the notification fallback only. |

### `slack.update_message`

Edit a message Valet previously posted. Only works on messages sent by Valet itself — Slack rejects bot edits to anyone else's messages. Use the `channel` and `ts` returned by `send_message` / `dm_owner` / `dm_user`. The new text fully replaces the previous content (including block-formatted content from a long `send_message`).

| Param | Type | Required | Description |
| --- | --- | --- | --- |
| `channel` | string | Yes | Channel ID (`C...`, or `D...` for DMs) returned when the message was sent. Channel names are not accepted — use the ID from the send result. |
| `ts` | string | Yes | Timestamp (`ts`) of the message to edit, as returned by `send_message`/`dm_owner`/`dm_user` (e.g. `"1780887543.189519"`). |
| `text` | string | Yes | New message body — fully replaces the previous content. Supports Slack mrkdwn: bold `*text*`, italic `_text_`, code `` `code` ``, links `<url\|label>`. |

### `slack.delete_message`

Delete a message Valet previously posted. Only works on messages sent by Valet itself — Slack rejects bot deletes of anyone else's messages. Irreversible: prefer `update_message` when the content just needs correcting.

| Param | Type | Required | Description |
| --- | --- | --- | --- |
| `channel` | string | Yes | Channel ID (`C...`, or `D...` for DMs) the message was posted to. |
| `ts` | string | Yes | Timestamp (`ts`) of the message to delete, as returned by `send_message`/`dm_owner`/`dm_user`. |

### `slack.list_channels`

List Slack channels. By default lists only channels the bot has joined. Set scope to "all" to discover all public channels in the workspace. Use prefix to filter by channel name prefix (e.g. "eng-" or "team-").

| Param | Type | Required | Description |
| --- | --- | --- | --- |
| `scope` | enum(joined,all) | No | Which channels to list: "joined" (default) = bot member channels, "all" = all public channels |
| `prefix` | string | No | Filter channels whose name starts with this prefix |

### `slack.read_history`

Read recent messages from a Slack channel the bot has joined. Use list_channels to get channel IDs. Each message ts can be used as thread_ts for replies. Use oldest/latest to narrow to a time window.

| Param | Type | Required | Description |
| --- | --- | --- | --- |
| `channel` | string | Yes | Channel ID (C...) |
| `limit` | number | No | Max messages per page (default 100, max 200) |
| `cursor` | string | No | Pagination cursor from a previous response's next_cursor |
| `oldest` | string | No | Only messages after this time. Accepts ISO-8601 datetime (e.g. "2026-05-19T00:00:00-07:00") or Unix timestamp (e.g. "1774000000.000000"). Include a timezone offset for date-boundary accuracy — bare dates like "2026-05-19" resolve to midnight UTC. Inclusive. |
| `latest` | string | No | Only messages before this time. Accepts ISO-8601 datetime (with timezone offset) or Unix timestamp. Inclusive. Defaults to now. |
| `filter` | string | No | Case-insensitive keyword filter applied client-side. Only messages whose text contains this substring are returned. Pagination still advances through all messages — use has_more/next_cursor to continue. |
| `threads_only` | boolean | No | When true, only return messages that have thread replies (reply_count > 0). Useful for finding discussions in noisy alert channels. |
| `include_subtypes` | boolean | No | When true, include system messages (joins, topic changes, etc.). Default false — only human/bot conversation. |

### `slack.update_usergroup`

Update Slack user group metadata such as name, handle, description, or default channels. Use `slack.add_usergroup_users` and `slack.remove_usergroup_users` for membership changes.

| Param | Type | Required | Description |
| --- | --- | --- | --- |
| `usergroup` | string | Yes | User group ID (`S...`) |
| `name` | string | No | New user group name. Must be unique among user groups. |
| `handle` | string | No | New mention handle without `@`. Must be unique among channels, users, and user groups. |
| `description` | string | No | Short user group description. Pass an empty string to clear it. |
| `channels` | string[] | No | Default channel IDs for this user group. Pass an empty array to clear defaults. |
| `additional_channels` | string[] | No | Additional channel IDs where members can add the user group. Pass an empty array to clear. |
| `enable_section` | boolean | No | Show this user group as a sidebar section for all group members. Requires at least one default channel. |
| `team_id` | string | No | Encoded team ID. Required only when using an org-level token. |

### `slack.add_usergroup_users`

Idempotently add users to a Slack user group. Existing members are skipped; Slack is only updated when membership changes. This wraps Slack's replace-all membership API, so avoid concurrent membership updates to the same user group.

| Param | Type | Required | Description |
| --- | --- | --- | --- |
| `usergroup` | string | Yes | User group ID (`S...`) |
| `users` | string[] | Yes | Slack user IDs (`U...` or `W...`) to add |
| `team_id` | string | No | Encoded team ID. Required only when using an org-level token. |

### `slack.remove_usergroup_users`

Idempotently remove users from a Slack user group. Missing members are skipped. Refuses to remove the final member because Slack requires disabling the group instead. This wraps Slack's replace-all membership API, so avoid concurrent membership updates to the same user group.

| Param | Type | Required | Description |
| --- | --- | --- | --- |
| `usergroup` | string | Yes | User group ID (`S...`) |
| `users` | string[] | Yes | Slack user IDs (`U...` or `W...`) to remove |
| `team_id` | string | No | Encoded team ID. Required only when using an org-level token. |
