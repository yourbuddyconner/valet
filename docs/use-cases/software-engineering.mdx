title: Software Engineering
description: Use Valet for repository work, PR review, issue triage, QA loops, and release follow-up.
---

Valet can work like an engineering teammate with a hosted workspace. It can inspect a repository, use connected tools, prepare changes, run checks, and keep a task alive in the background.

Typical work includes:

- Reviewing pull requests and leaving actionable feedback.
- Triaging issues from GitHub, Linear, Slack, or a scheduled workflow.
- Preparing code changes in an isolated workspace.
- Running tests, collecting logs, and preparing a summary.
- Splitting large investigations into smaller tasks.
- Asking for approval before risky operations.

## Product surfaces involved

- [Run tasks](/product/tasks) keeps each request, review, and result together.
- [Workspaces](/product/workspaces) explains where the work happens.
- [Memory](/product/memory) helps Valet reuse useful preferences and project notes.
- [Approvals](/product/approvals) control sensitive tools.
- [Connect tools](/product/tools) connects GitHub, Linear, Slack, and other systems.
