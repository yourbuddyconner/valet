Start with one real piece of work. Valet is easiest to understand when the task is clear, bounded, and easy to review.

## Before you start

You need:

- A Valet account.
- The web app or a connected channel.
- Any tools Valet should use for the task.
- A clear idea of which actions should wait for approval.

## Run your first task

<Steps>
  <Step title="Open Valet">
    Start in the web app. It gives you the clearest view of the task, progress, approvals, and result.
  </Step>
  <Step title="Ask for a specific outcome">
    Use a request like: "Summarize the open Linear issues assigned to me and suggest the top three to handle."
  </Step>
  <Step title="Add missing information">
    If Valet needs a tool, document, project, account, or channel it cannot reach yet, connect it or ask an admin to enable it.
  </Step>
  <Step title="Review before Valet acts">
    If Valet wants to send, write, delete, merge, or change something important, review the approval prompt before continuing.
  </Step>
  <Step title="Use the result">
    Read the summary, inspect what changed, and send a follow-up if the task needs more work.
  </Step>
</Steps>

## Good first tasks

Good first tasks are easy to inspect:

- Summarize a set of issues, messages, or documents.
- Draft a follow-up plan from connected tools.
- Prepare a checklist for a recurring process.
- Research account details before a meeting.
- Triage a small queue and ask before writing updates.

## Save what worked

After a successful first task:

- Turn repeated steps into a [workflow](/product/workflows).
- Capture useful instructions as a [skill](/product/skills-and-personas).
- Connect more [tools](/product/tools) if Valet needed information it could not reach.
- Adjust [approvals](/product/approvals) if the task prompted too often or too little.
