## What users may see

Depending on the task, a workspace can show:

- The conversation with Valet.
- Progress updates and questions.
- Approval prompts.
- Drafts, summaries, files, or notes.
- Links to records Valet read or changed.
- Software tools when the task is engineering work.

## How to get better results

- Tell Valet what outcome you want.
- Mention the systems or documents it should use.
- Connect missing tools before asking for multi-system work.
- Keep sensitive work private unless collaboration is needed.
- Use approvals for anything that writes, sends, deletes, merges, or changes customer-facing information.

## Workspace examples

| Work | What Valet keeps together |
| --- | --- |
| Weekly operations summary | Source messages, notes, draft summary, approvals, and final Slack update. |
| Customer follow-up | Account notes, open issues, draft response, and created tasks. |
| Pull request review | Repository notes, risks, suggested changes, and final summary. |
