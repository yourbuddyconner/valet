Manage repositories, branches, pull requests, issues, comments, checks, and webhooks.

## Connection summary

| Field | Value |
| --- | --- |
| Service slug | `github` |
| Display name | GitHub |
| Auth type | `oauth2` |
| Supported entities | `repositories`, `issues`, `pull_requests`, `commits` |

## Setup

OAuth 2.0 — the user connects this integration through Valet's connection screen, which redirects to the provider's OAuth consent flow and stores the resulting access token in the per-user credential store.

Connect from the **Integrations** page in the sidebar — click **Connect Integration**, pick the service, and approve the OAuth consent screen. (Self-hosting Valet? [OAuth setup](/oauth-setup) covers provider credential configuration.)

### `github.get_workflow_run`

Get details of a workflow run including jobs, steps, and check annotations. Use this to triage failures before fetching logs.

| Param | Type | Required | Description |
| --- | --- | --- | --- |
| `owner` | string | Yes | Repository owner |
| `repo` | string | Yes | Repository name |
| `run_id` | number | Yes | Workflow run ID |

### `github.get_job_logs`

Get log output from a specific workflow job. Parses logs into steps, filters to failed steps by default, strips noise, and truncates. Use after get_workflow_run to read failure details.

| Param | Type | Required | Description |
| --- | --- | --- | --- |
| `owner` | string | Yes | Repository owner |
| `repo` | string | Yes | Repository name |
| `job_id` | number | Yes | Job ID (from get_workflow_run response) |
| `failed_only` | boolean | No | Only return failed step output (default: true) |
| `step_name` | string | No | Filter to a specific step by name (partial match) |
| `tail_lines` | number | No | Max lines per step, tail-biased (default: 500) |
| `include_timestamps` | boolean | No | Include ISO timestamp prefixes (default: false) |

### `github.rerun_workflow`

## Triggers

GitHub ships an event trigger source that can fan webhook events into Valet workflows. Supported event types:

- `push`
- `pull_request`
- `issues`
- `issue_comment`
- `create`
- `delete`
- `release`
- `workflow_run`
- `check_run`
- `check_suite`
- `status`
- `ping`
