A task thread is where one piece of work happens in Valet. It is the place you use to ask for the work, watch progress, answer questions, approve sensitive steps, and collect the result.

<MascotNote

>
  Think of each task as a focused workroom. Give Valet one job, keep the relevant discussion there, and use follow-up messages when the work needs a nudge.
</MascotNote>

## Start a task

Start a new session when you want Valet to own a specific outcome.

Good requests include:

- "Summarize the open Linear issues assigned to me and suggest the top three to handle."
- "Review this pull request and call out risks before I merge it."
- "Draft a customer follow-up using the latest account notes."
- "Check the weekly operations queue and prepare a summary for Slack."

Avoid asking for broad, open-ended work like "fix everything" or "run the whole launch." Break that into smaller sessions or a workflow.

## Keep the work inspectable

A session should make it easy to answer:

- What did I ask Valet to do?
- What has Valet done so far?
- Is anything waiting on me?
- Which tools did Valet use?
- What changed?
- What should happen next?

If the answer is not clear, ask Valet to summarize progress before continuing.

## Share or hand off a session

Some work is private. Some work needs another person. Share a session when someone else needs to review, collaborate, or take over the result.

Use private tasks for personal notes, sensitive customer information, account setup, or work that only you should approve.

## Pause and continue

Valet can keep the history of a task so you can come back later. If the work is not done, send a follow-up message in the same thread instead of starting over.

For recurring work, create a [workflow](/product/workflows) so Valet can run the same pattern again.
