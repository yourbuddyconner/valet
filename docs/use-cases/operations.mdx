# Operations

Valet can handle recurring operational work when it has the right integrations, workflow definition, and approval policy. A user or organization orchestrator can receive requests, keep context, start sessions, and route the result back to the right channel.

Typical work includes:

- Running scheduled checks and posting summaries.
- Responding to Slack or Telegram requests.
- Pulling context from connected tools before taking action.
- Preparing drafts, reports, or follow-up tasks.
- Waiting for approval before actions that change external systems.
- Preserving memory about preferences and recurring procedures.

## Product surfaces involved

- [Triggers and workflows](/product/triggers-and-workflows) define recurring or event-driven work.
- [Channels](/product/channels) decide where requests and replies happen.
- [Orchestrators and memory](/product/orchestrators-and-memory) coordinate work over time.
- [Policies and approvals](/product/policies-and-approvals) keep control explicit.
- [Integrations](/product/integrations) define the tool surface Valet can reach.

<Note>
  Operations use cases are usually less about code and more about durable context, channel routing, and safe tool access.
</Note>
