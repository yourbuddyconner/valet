# Workflows

Workflows are repeatable ways to ask Valet to do work. They are useful when a task happens often, needs a consistent process, or should start from a schedule or external event.

## When to use a workflow

Use a workflow when the work has a repeatable shape:

- Run a weekly summary.
- Triage a queue each morning.
- React to a webhook from another system.
- Collect information from several tools before drafting a response.
- Pause for approval before a write, send, delete, merge, or other sensitive action.

Do not use a workflow when the task is vague or one-off. Start a normal session instead.

## Ways workflows start

| Start method | Use it for |
| --- | --- |
| Manual | A person chooses when the workflow runs. |
| Schedule | Recurring work, such as daily checks or weekly summaries. |
| Webhook | External systems that should hand work to Valet after an event. |

## Human review

Workflows can include approval moments. When Valet reaches one, the run pauses until an authorized user approves or denies the next step.

This matters for actions such as:

- Sending messages or emails.
- Creating, editing, or deleting records.
- Changing customer-facing information.
- Running operational actions that affect another system.
- Making code or repository changes.

## What users should inspect

For each workflow run, users should be able to answer:

- Why did this run start?
- What inputs did it receive?
- What did Valet do?
- Did it wait for approval?
- What changed?
- What should happen next?
