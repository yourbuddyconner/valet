# Orchestrators and Memory

An orchestrator is Valet's persistent coordinator. It remembers useful context, routes incoming work, and decides when a task should become its own session.

Think of it as the coworker you come back to between tasks.

## What an orchestrator does

An orchestrator can:

- Answer questions using remembered context.
- Receive work from channels or schedules.
- Decide whether to respond directly or start a focused session.
- Keep track of delegated work.
- Receive updates from child sessions.
- Apply your preferences to future tasks.

## What memory is for

Memory should store durable information that helps Valet work better over time:

- Communication preferences.
- Recurring procedures.
- Project or account context.
- Tool choices and operating norms.
- Decisions that should be remembered.
- Notes about workflows or repeated tasks.

Memory is not for every transient status update. Routine progress belongs in the session conversation.

## User and organization context

Valet can have personal context and organization context. Personal context helps with your preferences and work. Organization context helps teams share procedures, tools, and workflows.

Admins should decide what can be shared, what stays personal, and which channel surfaces can route into personal or organizational orchestrators.

## Delegating work

When a larger request needs focused execution, the orchestrator can delegate part of the work into a child session. This keeps the main conversation clear while the focused task has its own workspace, status, and result.
