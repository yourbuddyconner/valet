title: Models and reasoning
description: Pick which language model Valet uses, and understand reasoning behavior.

Valet runs on language models from providers like Anthropic, OpenAI, and Google. You can usually choose which model to use for a session, and your admin decides which models are available to your org.

## How to pick a model

There are two places you can set the model:

- **For one message — the `/model` slash command.** Type `/model` in the chat input to open a searchable picker grouped by provider. Pick a model and your next message will use it.
- **For all your sessions — model preferences in settings.** Open user settings, choose the models you want, and Valet will use those by default in new sessions. You can list multiple — the first one is the primary; the rest are used as fallbacks if the primary is unavailable.

Admins can restrict which models appear in the picker via org-level model preferences. If a model you expect is missing, ask your admin.

## What "reasoning" means

Some models — Anthropic's Claude family, OpenAI's o-series, and others — can do internal reasoning or "thinking" before they answer. The model spends extra tokens working through the problem privately, then produces the final response.

You'll see this in the UI when Valet shows a "thinking" status during a longer response. The reasoning itself is not shown to you, but the final answer reflects it.

Reasoning helps for:

- Multi-step planning and code changes.
- Debugging and root-cause analysis.
- Anything where the first guess is often wrong and a second pass helps.

Reasoning costs extra tokens and takes longer, so it's not always the right choice for short, simple requests.

## Switching mid-conversation

You can switch models in the middle of a conversation. Use `/model` to pick a new one for your next message; the conversation history continues uninterrupted. This is useful when:

- You started on a fast/cheap model and want a stronger one for a hard subtask.
- You want to compare how different models handle the same prompt.
- A model is hitting a quota or error and you want to fail over to a sibling model.

If a model fails (quota, provider outage, transient error), Valet will try the next model in your preferences list before reporting the failure.

## When a model gives a wrong or weird answer

- Try the same prompt on a different model via `/model` — the issue is sometimes model-specific.
- Be more explicit in your request. Reasoning models reward clear, specific prompts.
- For factual or date-sensitive answers, see [troubleshooting](/product/troubleshooting#valet-got-the-date-or-day-of-the-week-wrong).

## Cost and quota

Different models cost different amounts. If your org is hitting credit or quota limits, see [billing](/product/billing) for how that is managed, and [troubleshooting](/product/troubleshooting#credit-balance-too-low-or-quota-exceeded) for what to do when prompts fail with a quota error.
