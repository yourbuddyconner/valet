A session is where one piece of work starts in Valet. It is the conversation you use to ask for the work, watch progress, answer questions, approve sensitive steps, and collect the result.

  Think of each session as a focused workroom. Give Valet one job, keep the relevant discussion there, and use follow-up messages when the work needs a nudge.
</MascotNote>

## Two layers of work

Most users only need the session itself. Internally, this is the orchestrator session: Valet's always-on coworker owns the conversation, keeps it readable, and decides whether deeper work needs to happen somewhere else.

Some work also needs a child session. A child session is a separate workspace Valet starts when the work needs stronger isolation, such as code changes, files, a browser, a terminal, or a focused subtask that should not mix with the main conversation.

| Layer | What it isolates | When it matters |
| --- | --- | --- |
| Orchestrator session | Conversation, participants, approvals, updates, and result. | Every piece of work. |
| Child session | Workspace, files, tools, running work, and software state. | Engineering work, long-running research, or focused subtasks. |

Valet decides when a child session is useful. You can still follow the work from the main session.

## Start a session

Start a new session when you want Valet to own a specific outcome.

Good requests include:

- "Summarize the open Linear issues assigned to me and suggest the top three to handle."
- "Review this pull request and call out risks before I merge it."
- "Draft a customer follow-up using the latest account notes."
- "Check the weekly operations queue and prepare a summary for Slack."

Avoid asking for broad, open-ended work like "fix everything" or "run the whole launch." Break that into smaller sessions or a workflow.

## Keep the work inspectable

A session makes it easy to answer:

- What did I ask Valet to do?
- What has Valet done so far?
- Is anything waiting on me?
- Which tools did Valet use?
- What changed?
- What should happen next?

If the answer is not clear, ask Valet to summarize progress before continuing.

## Share or hand off a session

Some work is private. Some work needs another person. Share a session when someone else needs to review, collaborate, or take over the result.

Use private sessions for personal notes, sensitive customer information, account setup, or work that only you should approve.

## Pause and continue

Valet can keep the history of a session so you can come back later. If the work is not done, send a follow-up message in the same thread instead of starting over.

For recurring work, create a [workflow](/product/workflows) so Valet can run the same pattern again.
