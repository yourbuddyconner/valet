title: Connect Tools
description: Connect the systems Valet should read from or act in.
---

Integrations are the tools Valet can use. Connect them when Valet needs to gather information, prepare updates, create tasks, send messages, or complete work across systems.

## What to connect

Common tool categories include:

- Issue trackers and project tools.
- Chat and messaging tools.
- Email and calendar.
- Documents, drives, and spreadsheets.
- Repositories and engineering tools.
- Customer systems.
- Monitoring and incident tools.
- Finance or billing systems.

The exact list depends on what your organization enables.

## Before connecting a tool

Decide:

- Who should be allowed to connect it?
- Should the connection be personal or shared by the team?
- What should Valet be allowed to read?
- What should Valet be allowed to change?
- Which actions should require approval?
- Where should Valet report results?

## Good first integrations

Start with the tools that hold the information for your first real tasks:

- Use GitHub or Linear for engineering triage.
- Use Slack for team requests and updates.
- Use Gmail and Google Workspace for customer or operations work.
- Use Notion or Docs for internal procedures.

Add more tools after the first workflows are clear.

## Tool safety

Valet should not need passwords, keys, or other secrets in the conversation. Connect tools through approved setup screens, then control actions with [approvals](/product/approvals).

## What users should expect

When Valet uses a connected tool, the session should make it clear:

- Which tool was used.
- What Valet read or prepared.
- Whether anything changed.
- Whether a human approved the change.
