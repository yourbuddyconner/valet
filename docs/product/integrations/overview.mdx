title: Integrations
description: Reference for every external integration shipped with Valet, including auth type, actions, and triggers.
---

Valet ships first-class integrations as plugins under `packages/plugin-*`. Each plugin contributes one or more actions an agent can call, an optional trigger source for inbound events, and a provider definition that controls how credentials are stored.

Two integration shapes are in use today:

- **Inline action plugins** — actions are defined in Valet's own source with explicit names, parameters, and risk levels. The pages below list every action and its params.
- **MCP-proxied plugins** — actions are forwarded to a remote [Model Context Protocol](https://modelcontextprotocol.io) server. The plugin owns auth and the default risk level; the upstream MCP server owns the live action catalog. Agents can enumerate the live actions with `list_tools`.

All actions, regardless of shape, are subject to the same [approvals](/product/approvals) and [policy](/engineering/policy-engine) layers.
