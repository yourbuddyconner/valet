# Approvals and Controls

Valet can use connected tools, but sensitive actions should stay under human control. Policies and approvals decide when Valet can act automatically, when it must ask, and when it cannot proceed.

## Why approvals matter

Approvals protect actions such as:

- Sending external messages.
- Creating, editing, or deleting records.
- Changing customer-facing information.
- Modifying repositories or production systems.
- Accessing sensitive connected tools.
- Running workflow steps that have business impact.

## Approval choices

When Valet asks for approval, a user may see choices such as:

| Choice | Meaning |
| --- | --- |
| Allow | Run this one action. |
| Allow for session | Allow the same action during the current session. |
| Always allow | Remember this exact action as allowed. |
| Cancel | Do not run the action. |

Organizations can restrict which choices are available.

## Admin controls

Admins can set rules for services, actions, or risk levels. A strict organization rule should override a user's personal preference.

Use admin policy for actions that should always require review or never be allowed.

## User preferences

Users may be able to customize their own approval experience for lower-risk actions. This helps Valet move faster without weakening organization-level controls.

## Auditability

Valet should preserve enough context to explain:

- What action was requested.
- Which service or integration was involved.
- Who approved or denied it.
- Whether the action ran.
- Which policy or preference shaped the decision.
