# Software Engineering

Valet can work like an engineering teammate with a hosted workstation. It can inspect a repository, run commands, use a browser, open VS Code, call connected tools, and keep a task alive in the background.

Typical work includes:

- Reviewing pull requests and leaving actionable feedback.
- Triaging issues from GitHub, Linear, Slack, or a scheduled workflow.
- Making code changes in a sandboxed workspace.
- Running tests, collecting logs, and preparing a summary.
- Spawning child sessions for parallel investigation.
- Asking for approval before risky operations.

## Product surfaces involved

- [Sessions](/product/sessions) own the live task state.
- [Workstations and sandboxes](/product/workstations-and-sandboxes) provide the execution environment.
- [Orchestrators and memory](/product/orchestrators-and-memory) keep durable context across work.
- [Policies and approvals](/product/policies-and-approvals) control sensitive tools.
- [Integrations](/product/integrations) connect GitHub, Linear, Slack, and other systems.
