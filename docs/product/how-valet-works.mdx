---
title: How Valet Works
description: A simple view of how Valet turns a request into useful work.
---

Valet turns a request into useful work by following a clear path: understand the job, gather what it needs, do the work, ask before sensitive actions, and return the result.

## The flow

<Steps>
  <Step title="You ask for work">
    Start from the web app, Slack, Telegram, a schedule, or a saved workflow.
  </Step>
  <Step title="Valet gathers what it needs">
    Valet uses the conversation, connected tools, remembered preferences, and any details you provide.
  </Step>
  <Step title="Valet works in a sandbox when needed">
    Work that runs code or handles files happens in an isolated [sandbox](/product/workspaces) — a real file system, editor, and terminal scoped to the task. Everything else stays in the task thread.
  </Step>
  <Step title="Valet uses connected tools">
    If you have connected the right systems, Valet can read from them or prepare changes in them.
  </Step>
  <Step title="Valet asks before sensitive actions">
    Sending, writing, deleting, merging, or changing important information can require approval.
  </Step>
  <Step title="You get the result">
    Valet returns a summary, draft, update, file, task, or next step in the place where the work belongs.
  </Step>
</Steps>

## What stays visible

Every task shows you:

- What Valet is working on.
- What it has found so far.
- What it needs from you.
- Which tools it used.
- What changed.
- Where to continue the work.

## What you control

You can control:

- Which tools Valet can use.
- Which actions need approval.
- Where Valet replies.
- Who can see or join a task.
- Which instructions, skills, or workflows Valet should follow.
