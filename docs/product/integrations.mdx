# Integrations

Integrations are the tools Valet can use. They let Valet gather context, create drafts, update records, send messages, and complete work across systems.

Valet should only use an integration when a user or admin has connected it and policy allows the action.

## What integrations are for

Use integrations when Valet needs to:

- Read context from another system.
- Create or update work items.
- Draft or send messages.
- Inspect repository or issue state.
- Coordinate with calendars, documents, sheets, or notes.
- Run customer, operations, or engineering workflows.

## Current integration areas

Valet can be connected to systems such as:

- GitHub.
- Slack.
- Telegram.
- Gmail.
- Google Calendar.
- Google Drive, Docs, and Sheets.
- Linear.
- Notion.
- Stripe.
- Sentry.
- Cloudflare.
- Figma.
- Grafana Cloud.
- Pylon.
- Typefully.
- Custom MCP services.

The exact catalog depends on what is enabled in the deployment.

## Credentials and safety

Users and admins connect accounts so Valet can act through approved paths. Valet should not expose raw credentials to the agent. It should perform actions through controlled integration calls, policy checks, and audit records.

## How to think about access

For each integration, decide:

- Who can connect it?
- Is it personal, team-wide, or organization-wide?
- Which actions are read-only?
- Which actions require approval?
- Which actions should be denied?
- Where should results be reported?
