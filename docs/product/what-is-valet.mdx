# What is Valet?

Valet lets a user delegate work to an AI coworker that has its own hosted workstation. The coworker can keep context, use connected tools, coordinate follow-up work, ask for approvals, and report progress through the web UI or connected channels.

When the task is engineering work, the workstation becomes a full development environment: the agent can clone repositories, inspect code, run commands, use browser and terminal access, and open pull requests. When the task is operational work, the same workstation model lets the agent use integrations, workflows, and channel context without pretending everything is a coding session.

## Product model

Valet is organized around a few product concepts:

- **Workstations** are hosted environments where agents do work. For coding, this includes editor, browser, terminal, repository, and runtime access.
- **Sessions** are individual units of work. They hold conversation, participants, runtime state, approvals, and task-specific context.
- **Orchestrators** are persistent coordinator agents. They remember preferences, route messages, manage task context, and decide when to spawn child sessions.
- **Workflows** are repeatable automations that can run manually, on a schedule, or from a webhook.
- **Plugins** connect Valet to tools, channels, skills, personas, and external services.

## Technical architecture

For technical readers, each session is coordinated by Cloudflare infrastructure and executed inside Modal:

- The **React web app** lets users create sessions, chat with agents, watch streaming output, open workspace iframes, and manage settings.
- The **Cloudflare Worker** handles API routes, auth, integration configuration, webhooks, and Durable Object entry points.
- A **SessionAgent Durable Object** owns live session state, WebSocket connections, prompt queues, message streaming, hibernation, restoration, and participant access.
- The **Modal backend** creates, restores, hibernates, and terminates sandboxes.
- The **Runner** inside each sandbox bridges the Durable Object to OpenCode, manages workflow execution, and exposes the sandbox auth gateway.
- **Plugins** supply action catalogs, channel transports, skills, personas, and sandbox tools.

## The user experience

A user can:

- Start a manual session from the web UI.
- Ask their orchestrator to plan, answer, or spawn child sessions.
- Connect Slack or Telegram and route messages into Valet.
- Configure GitHub and other integrations so agents can work with external systems.
- Define workflows that run manually, on a schedule, or from webhooks.
- Approve or deny risky tool calls, with optional session or persistent overrides.
- Hibernate sessions to save resources and restore them later.
